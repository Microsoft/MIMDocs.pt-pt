# <a name="management-agent-run-error-codes"></a>Agente de gestão executar códigos de erro
As tabelas que se seguem listam os códigos de erro da interface de utilizador do Gestor de Serviços de Sincronização no Microsoft Identity Manager (MIM) 2016. São fornecidas descrições detalhadas para explicar cada erro.

## <a name="connection-errors"></a>Erros de ligação

|Erro | Descrição |
|----|----|  
| ligação&#8209;falhou         | A ligação ao diretório ligado falhou por outra razão que não a autenticação. Por exemplo, a rede não está disponível ou o servidor-alvo está offline. |
| deixou cair&#8209;ligação        | A ligação entre o agente de gestão e o diretório ligado já não existe. Em muitos casos, o agente de gestão tenta reconectar-se com o diretório conectado.  |
| autenticação&#8209;falhada     | A autenticação não é possível utilizando as credenciais fornecidas.  |
| falhou&#8209;permissão         | Não existem direitos suficientes para aceder a um contentor no diretório ligado. O erro é esperado apenas para agentes de gestão do Protocolo de Acesso ao Diretório Leve (LDAP) que procuram diferentes contentores de diretórios conectados.  |
| pesquisa de&#8209;falhou            | Uma procura de contentores ou mesa falhou com um erro inesperado. |
| aviso&#8209;nenhuma marca de água&#8209;      | O agente de gestão não consegue ler a marca de água quando faz uma importação completa. O erro é esperado apenas para o agente de gestão do Sun ONE Directory Server 5.1 (anteriormente iPlanet Directory Server) quando a configuração inicial do agente de gestão estiver concluída e o diretório ligado tiver o registo de alteração ativado. Mais tarde, quando o registo de alteração do diretório ligado é desligado, se a configuração do agente de gestão não for atualizada, o aviso ocorre quando uma importação completa é feita. |
| nenhuma&#8209;começar&#8209;&#8209;de partição | A partição LDAP originalmente configurada para este agente de gestão já não existe. O erro é devolvido quando a partição foi eliminada, ou a partição foi eliminada e recriada com o mesmo nome. Neste último caso, embora o nome seja o mesmo, a partição de erro GUID mudou. |

## <a name="discovery-errors"></a>Erros de descoberta

|Erro | Descrição |
|----|----|  
| falta&#8209;alteração&#8209;tipo      | Este erro é devolvido durante uma importação delta gerida por agentes de gestão de ficheiros e de base de dados, bem como o agente de gestão dos servidores de diretório Sun e Netscape, quando o valor da coluna tipo de alteração (adicionar, modificar, excluir) não estiver presente. |
| alteração&#8209;inválida&#8209;tipo      | Este erro é devolvido durante uma importação delta gerida por agentes de gestão de ficheiros e bases de dados, bem como o agente de gestão dos servidores de diretório Sun e Netscape quando o valor da coluna de tipo de alteração não corresponde à lista de tipos de alteração válidos. Também é devolvido de um Formato de Intercâmbio de Dados LDAP (LDIF) importado quando um campo de tipo de alteração está presente e tem um valor diferente de adicionar. |
| &#8209;&#8209;alteração&#8209;tipo | Este erro é devolvido durante uma importação delta gerida por agentes de gestão de servidores de diretórios Sun e Netscape quando há mais de um valor para o tipo de alteração. |
| precisa&#8209;objeto&#8209;cheio        | Este erro é devolvido durante uma corrida de importação delta de um agente de gestão baseado em ficheiros ou quando retoma de um agente de gestão baseado em ficheiros. Indica que o agente de gestão submeteu uma modificação num objeto que não pode ser localizado no espaço do conector. O motor de sincronização está a solicitar os valores atuais de todos os atributos no objeto. Uma vez que se trata de uma importação de um ficheiro, essa informação não está disponível. Uma importação completa deve resolver este problema. |
| falta&#8209;dn              | Este erro é devolvido para agentes de gestão baseados em ficheiros (isto é, agentes de gestão para LDIF, DSML ou ficheiros planos com atributos de nome de domínio configurados) quando não há valor de nome de domínio. Isto também é devolvido para um registo de alteração do servidor de diretório Sun ONE corrompido onde o atributo de nome de domínio está em falta. Indica que o agente de gestão podia ler o elemento e analisá-lo, mas não havia valor de nome de domínio para o objeto. |
| dn&#8209;não&#8209;ldap&#8209;conformante   | Este erro é devolvido quando um agente de gestão de LDAP, LDIF, DSML ou um ficheiro plano com um atributo de nome de domínio configurado reporta um valor de nome de domínio que não está em conformidade com a especificação LDAP. |
| inválido&#8209;dn              | Este erro é devolvido quando um agente de gestão informa que um nome de domínio não cumpre uma restrição FIM, que inclui:   <br/><ul><li>Um ou mais caracteres que não são permitidos pela FIM.</li><li>Um nome distinto relativo vazio (também conhecido como RDN). </li><li>Um nome distinguido relativo que exceda o máximo para FIM. </li><li>O número de níveis de hierarquia do nome de domínio excede o máximo para FIM.</li></ul> |
| componente&#8209;de âncora&#8209;em falta | Este erro é devolvido por agentes de gestão de ficheiros e de bases de dados, bem como pelo agente de gestão dos servidores de diretório Sun e Netscape, quando a âncora não pode ser construída porque um ou mais atributos de regra de construção de âncora não têm valores. |
| multi&#8209;valorizados&#8209;componente de&#8209;âncora | Este erro é devolvido pelo agente de gestão para servidores de diretório Sun e Netscape se não conseguirem construir a âncora porque um atributo de regra de construção de âncora tem mais de um valor. |   
|âncora&#8209;muito&#8209;longo | Este erro é devolvido por agentes de gestão de ficheiros e de bases de dados, bem como pelo agente de gestão dos servidores de diretório Sun e Netscape, quando a construção da âncora produz uma âncora que excede o limite máximo de tamanho para FIM. |
|duplicar&#8209;objeto | Este erro é devolvido em importações completas por agentes de gestão de ficheiros e de bases de dados quando um objeto com a mesma âncora já foi reportado ao motor de sincronização durante esta execução.<br/>**Nota:** A obsolescência dos objetos espaciais do conector só ocorre se o passo de execução atual tiver terminado com Sucesso, Completo com falha de sincronização, completo com aviso ou completo com transitório. |
|desaparecimento&#8209;objeto&#8209;classe | Este erro é devolvido por um agente de gestão baseado em ficheiros (isto é, um agente de gestão da DSML, LDIF ou um ficheiro plano com um atributo de classe de objetos configurado), ou pelo agente de gestão dos servidores de diretório Sun e Netscape, se houver um registo de alteração corrompido. Isto indica que o agente de gestão não pode ler um valor para o atributo da classe objeto. |
|tipo&#8209;objeto&#8209;desaparecido | Este erro é devolvido quando efetua um currículo de importação de um ficheiro de queda corrompido. Este erro não é encontrado durante o funcionamento normal.        |
|inaplicável-tipo de objeto  | Este erro é devolvido por um agente de gestão baseado em ficheiros quando lê um objeto que tem um conjunto de valores de classe de objetos que não podem ser correspondidos a nenhum dos mapeamentos do prefixo. |
|erro de&#8209;de parse | Este erro é devolvido pelo agente de gestão para servidores de diretório Sun e Netscape em modo delta e por agentes de gestão baseados em ficheiros quando não podem analisar uma entrada. O `<entry-number>` elemento (e na maioria dos casos `<line-number>` `<column-number>` e) está presente para ajudar a localizar o erro. O `<attribute-name>` elemento pode estar presente. O agente de gestão dos servidores de diretório Sun e Netscape termina a execução quando esta é encontrada. Os agentes de gestão baseados em ficheiros registam o erro de descoberta e continuam. |   
|ler&#8209;erro | Este erro é devolvido por agentes de gestão baseados em chamadas quando há um erro genérico a ler um determinado objeto. Isto geralmente causa o fim da corrida. O elemento de erro de fonte de dados ligado está presente, que pode utilizar para resolver o problema.    |
|encenação&#8209;erro | Este erro é devolvido pela maioria dos agentes de gestão. Indica que o motor de sincronização não conseguiu encenar o delta no espaço do conector. O servidor cria um registo de eventos que fornece informações sobre o problema e que pode ser usado para a resolução de problemas. A maioria dos agentes de gestão continua a importação quando o erro é registado, mas o agente de gestão da Sun e netscape delta runs para. As lacunas no processamento do registo de alteração podem causar um estado inconsistente no espaço do conector. Este erro não é encontrado durante o funcionamento normal.  |
|tipo&#8209;de modificação&#8209;inválido | Este erro é devolvido durante uma importação delta a um agente de gestão LDIF quando um tipo de modificação de nível de objeto não é um dos tipos padrão de modificação LDIF ou existe um `nonreplace` tipo de modificação na classe de objetos, tais como `add: objectclass` ou `delete: objectclass` .    |
|&#8209;tipos de modificação&#8209;&#8209;conflituosos        | Este erro é devolvido pelo agente de gestão LDIF indicando que foram encontrados diferentes tipos de modificação de nível de atributos no mesmo registo (neste caso é relatado o nome de atributo que produziu os tipos de conflito) ou são observados vários deltas LDIF no mesmo ficheiro, tais como:    <br/> [Exemplo de arquivo LDIF](./media/maerrorcodes/conflict-modification-types.jpg) |
|multi&#8209;único&#8209;desfasamento | Este erro é devolvido por um agente de gestão baseado em ficheiros quando reporta mais do que um valor acrescentado, ou mais de um valor apagar para um atributo que é definido no FIM como sendo um atributo de valor único. Este erro pode indicar que o esquema de origem de dados ligado que é armazenado com FIM está incorretamente especificado (agentes de gestão baseados em ficheiros) ou desatualizado com o esquema atual. Inclui um `<attribute-name>` elemento para dar o contexto do erro.    |
|valor&#8209;&#8209;&#8209;inválido | Este erro é devolvido por um agente de gestão baseado em chamadas quando é lido um valor de atributo que não está em conformidade com o tipo de atributo declarado no esquema. Inclui um `<attribute-name>` elemento para dar o contexto do erro.  |
|valor&#8209;base&#8209;inválido| Este erro é devolvido pelos agentes de gestão dos servidores de diretório LDIF, DSML, Sun e Netscape quando encontram uma cadeia base 64 inválida.     |
|valor&#8209;numérico inválido&#8209; | Este erro é devolvido por agentes de gestão baseados em ficheiros e pelo agente de gestão da LDAP quando não conseguem analisar um valor numérico. Inclui um `<attribute-name>` elemento para dar o contexto do erro. |
|valor&#8209;&#8209;inválido | Este erro é devolvido por agentes de gestão baseados em ficheiros e pelo agente de gestão da LDAP quando não conseguem analisar um valor Boolean. Inclui um `<attribute-name>` elemento para dar o contexto do erro.  |
|valor&#8209;de referência não&#8209;&#8209;conformante ldap&#8209; | Este erro é devolvido pelos agentes de gestão para LDAP, LDIF e DSML ou ficheiros planos (com atributo de nome de domínio configurado) quando um valor de nome de domínio não está em conformidade com a especificação LDAP. Esta mensagem de erro inclui um `<attribute-name>` elemento para dar o contexto do erro.|
|valor&#8209;de referência&#8209;inválido | Este erro é devolvido por um agente de gestão quando um nome de domínio não satisfaz os constrangimentos fim, que incluem:  <br/><ul><li>Um ou mais caracteres que não são permitidos pelo FIM</li><li>Um nome relativo vazio (também conhecido como RDN)  </li><li>Um nome distinguido relativo que exceda o máximo para FIM  </li><li>O número de níveis de hierarquia do nome de domínio excedeu o máximo para o FIM.</li></ul>  |
|tipo&#8209;de valor&#8209;não suportado | Este erro é devolvido pelo agente de gestão DSML ou LDIF quando o tipo de valor dado no ficheiro é incompatível com o tipo de atributo, incluindo:    <br/><ul><li>Um valor URI ou URL é dado para um atributo não-string ou para qualquer palavra-chave reservada, como dn, objectclass ou changetype.</li><li>É dado um valor base64 para o atributo de alteração.</li><li>Um valor de cadeia que contém caracteres não ASCII é dado para um atributo binário.</li></ul> |

## <a name="export-errors"></a>Erros de exportação

|Erro | Descrição |
|----|----|  
| cd&#8209;objeto&#8209;desaparecido         | Este erro é devolvido quando uma modificação de um objeto é exportada para a fonte de dados conectada, mas o objeto não pode ser encontrado na fonte de dados conectada. É devolvido apenas para agentes de gestão baseados em chamadas. A causa deste erro é que uma pessoa ou processo externo apagou um objeto da fonte de dados conectada fora do FIM.  |
| cd&#8209;objeto&#8209;existente        | Este erro é devolvido quando um adição é exportado para a fonte de dados conectada, mas o objeto já está presente na fonte de dados conectada. É devolvido apenas para agentes de gestão baseados em chamadas e agentes de gestão de bases de dados relacionais. |
