# <a name="management-agent-run-error-codes"></a>Códigos de erro de execução do agente de gestão

As tabelas seguintes incluem códigos de erro que podem aparecer na interface de utilizador de Gestor do serviço de sincronização no Microsoft Identity Manager (MIM) 2016, bem como descrições para cada um desses erros.

## <a name="connection-errors"></a>Erros de ligação

| Erro                  | Descrição                                    |
|--------------------------|--------------------------------------------|
| Falha na ligação de =         | A ligação para o diretório ligado falhou por um motivo que não sejam de autenticação. Por exemplo, a rede não estiver disponível, ou o servidor de destino está offline.                                                                                                                                                                                                                                                                                                                      |
| ligação ignorados        | A ligação entre o agente de gestão e o diretório ligado já não existe. O agente de gestão tenta restabelecer a ligação para o diretório ligado em várias instâncias.                                                                                                                                                                                                                                                                                                         |
| Falha na autenticação     | Autenticação não é possível utilizar as credenciais fornecidas.                                                                                                                                                                                                                                                                                                                                                                                                                          |
| permissão falhou         | Direitos insuficientes para aceder a um contentor no diretório ligado. Este erro só é esperado para agentes de gestão de acesso protocolo LDAP (Lightweight Directory) pesquisar contentores de outro diretório ligado.                                                                                                                                                                                                                                                              |
| pesquisa de falha            | Procurar um contentor ou tabela falhou com um erro inesperado.                                                                                                                                                                                                                                                                                                                                                                                                                            |
| aviso-não-marca d'água      | O agente de gestão não é possível ler a marca d'água ao efetuar uma importação completa. Este erro é esperado apenas para o agente de gestão para Sun um diretório servidor 5.1 (anteriormente iPlanet servidor de diretório) quando a configuração do agente de gestão inicial está concluída e o diretório ligado tem registo de alterações ativado. Mais tarde, quando o registo de alterações de diretório ligado estiver desligado, se a configuração do agente de gestão não for atualizada, este aviso ocorre quando é efetuada uma importação completa. |
| não-start--a eliminação de partições | A partição LDAP configurada originalmente para este agente de gestão já não existe. Este erro é devolvido quando a partição foi eliminada ou a partição foi eliminada e recriada com o mesmo nome. No caso desta última opção, apesar do nome é o mesmo, a partição de erro GUID irá ter sido alterado.                                                                                                                                                                           |

## <a name="discovery-errors"></a>Erros de deteção

| Erro                   | Descrição                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
|-------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Falta--tipo de alteração      | Este erro é devolvido durante uma importação delta executada por baseados em ficheiros e base de dados agentes de gestão, bem como o agente de gestão para Sun e Netscape servidores de diretório, quando a alteração do tipo de valor de coluna (adicionar, modificar, eliminar) não está presente.                                                                                                                                                                                                                                          |
| tipo de alteração inválida      | Este erro é devolvido durante uma importação delta executada por baseados em ficheiros e tipos de alteração de agentes de gestão de base de dados, bem como o agente de gestão para servidores de diretório Sun e Netscape quando o valor de coluna de tipo de alteração não corresponde a lista de válido. Também é devolvido de uma importação completa de LDAP dados intercâmbio LDIF (formato) quando um campo de tipo de alteração está presente e tem um valor diferente de adicionar.                                                                                  |
| várias-valued--tipo de alteração | Este erro é devolvido durante uma importação delta executada por baseados em ficheiros e Sun e Netscape diretório servidores agentes de gestão quando mais do que um valor para o tipo de alteração está presente.                                                                                                                                                                                                                                                                                                        |
| necessário-full-objeto        | Este erro é devolvido durante uma execução de um agente de gestão baseada em ficheiros, ou quando a retomar a partir de um agente de gestão baseada em ficheiros a importação delta. Indica que o agente de gestão foi submetida uma modificação de um objecto que não é possível localizar no espaço de conector. O motor de sincronização está a pedir os valores atuais de todos os atributos do objeto. Uma vez que esta é uma importação de um ficheiro, essas informações não estão disponíveis. Uma importação completa deve resolver este problema. |
| dn em falta              | Este erro é devolvido para agentes de gestão baseada em ficheiros (ou seja, agentes de gestão para LDIF, DSML ou ficheiros simples com atributos de nome de domínio configurados) quando qualquer valor de nome de domínio. Este é também devolvido no caso de um registo de alterações de um servidor de diretório Sun danificado onde o atributo de nome de domínio está em falta. Indica que o agente de gestão foi possível ler o elemento e analisá-la, mas havia nenhum valor de nome de domínio para o objeto.                           |
| DN não-ldap compatível   | Este erro é devolvido quando um agente de gestão para o LDAP, LDIF, DSML, um ficheiro simples com um atributo de nome de domínio configurados relatórios ou um valor de nome de domínio que não está em conformidade com a especificação de LDAP.                                                                                                                                                                                                                                                                                |
| dn inválido              | Este erro é devolvido quando um agente de gestão que um nome de domínio não cumpre uma restrição FIM, que inclui:   <br/>&#176; Um ou mais caracteres que não são permitidas por FIM <br/>&#176; Um nome único relativo em branco (também conhecido como RDN) <br/>&#176; Um nome distinto relativo que excede o máximo de FIM <br/>&#176; O número de níveis hierárquicos, o nome de domínio excede o máximo de FIM                                                                                                                                                                                                                                                                                                                                                            |
| falta-âncora-componente | Este erro é devolvido pelo baseados em ficheiros e os agentes de gestão de base de dados, bem como o agente de gestão para Sun e Netscape servidores de acesso do, quando a âncora não pode ser construída porque um ou mais atributos de regra de construção de ancorar não dispõe de valores.   |
| várias-valued-âncora-componente | Este erro é devolvido pelo agente de gestão para Sun e Netscape servidores de diretório se estes não é possível construir a âncora porque um atributo de regra de construção de âncora tem mais do que um valor. |   
|âncora-demasiado-longa | Este erro é devolvido pelo baseados em ficheiros e o limite agentes de gestão de base de dados, bem como o agente de gestão para Sun e Netscape servidores de acesso do, quando a construção de âncora produz uma âncora que excede o tamanho máximo do FIM.|
|objeto de duplicado | Este erro é devolvido em importações completas por baseados em ficheiros e os agentes de gestão de base de dados quando um objeto com a mesma âncora já foi comunicado para o motor de sincronização durante a execução.|

>[!Note]
Obsolescence de objetos de espaço de conector ocorrerá apenas se a execução do passo atual foi concluída com êxito, concluída com falhas de sincronização, concluída com aviso ou concluído com transitório.

|Erro| Descrição |
|----|----|  
|falta--classe de objeto | Este erro é devolvido por um agente de gestão baseada em ficheiros (ou seja, um agente de gestão para DSML, LDIF ou um ficheiro simples com um atributo de classe de objeto configuradas) ou para o agente de gestão para Sun e Netscape servidores de diretório, se existir um registo de alterações está danificado. Isto indica que o agente de gestão não é possível ler um valor para o atributo de classe de objeto. |

|Erro de análise | Este erro é devolvido pelo agente de gestão para Sun e Netscape servidores com o diretório no modo de diferenças e por agentes de gestão baseada em ficheiros quando estes não é possível analisar uma entrada. O `<entry-number>` elemento (e na maioria dos casos `<line-number>` e `<column-number>`) estará presente para ajudar a localizar o erro. O `<attribute-name>` elemento poderá estar presente. O agente de gestão para servidores de diretório Sun e Netscape termina a executar quando este foi encontrado. Os agentes de gestão baseada em ficheiros de registo de erro de deteção e continuarem. |   
|Erro de leitura | Este erro é devolvido por agentes de gestão baseado na chamada quando existe um erro genérico ler um objeto específico. Isto, geralmente, faz com que terminação a execução. O elemento de erro da origem de dados ligada estiver presente, o que pode utilizar para resolver o problema.    |
|Erro de teste | Este erro é devolvido pela maioria dos agentes de gestão. Indica que o motor de sincronização não foi possível testar o delta no espaço de conector. O servidor cria um registo de eventos que fornece informações sobre o problema e que podem ser utilizados para resolução de problemas. A maioria dos agentes de gestão continuam a ser executada quando o erro é registado, mas o agente de gestão para Sun e Netscape diferenças é executada interrompe porque lacunas no processamento do registo de alterações pode estar fazer com que um estado inconsistente no espaço de conector de importação. Este erro não deve ser detetado durante o funcionamento normal.  |
|tipo de modificação inválida | Este erro é devolvido durante uma importação delta num agente gestão LDIF quando um tipo de nível de modificação do objeto não é um dos tipos de modificação de LDIF padrão ou existe um tipo de modificação nonreplace no objectclass, tais como adicionar: objectclass ou eliminar: objectclass.    |
|em conflito modificação-tipos        | Este erro é devolvido por gestão LDIF agente com a indicação de diferentes tipos de nível de modificação do atributo foram encontrado no mesmo registo (neste caso, é reportado o nome de atributo que produziu os tipos em conflito) ou vários deltas LDIF substituir se verificam no mesmo ficheiro, tal como:    <br/> [Exemplo de ficheiro LDIF](./media/maerrorcodes/conflict-modification-types.jpg) |
|várias single incompatibilidade | Este erro é devolvido por um agente de gestão baseada em ficheiros quando esta comunica mais do que um valor de adicionar ou eliminar mais de um valor para um atributo que está definido no FIM como sendo um atributo de valor único. Este erro poderá indicar que o esquema de origem de dados ligada que esteja armazenado com o FIM incorretamente especificado (agentes de gestão baseada em ficheiros) ou desatualizada com o esquema atual. Inclui um `<attribute-name>` elemento para dar o contexto do erro.    |
|valor de atributo inválido | Este erro é devolvido por um agente de gestão baseado na chamada quando um valor de atributo é leitura não está em conformidade com o tipo de atributo declarada no esquema. Inclui um `<attribute-name>` elemento para dar o contexto do erro.  |
|valor de base64 inválido| Este erro é devolvido pela agentes de gestão para LDIF, DSML e Sun e Netscape servidores de diretório, quando se deparam uma cadeia base64 inválido.     |
|valor de numérica inválida | Este erro é devolvido por agentes de gestão baseada em ficheiros e o agente de gestão para o LDAP quando estão não é possível analisar um valor numérico. Inclui um `<attribute-name>` elemento para dar o contexto do erro. |
|valor de booleano inválido | Este erro é devolvido por agentes de gestão baseada em ficheiros e o agente de gestão para o LDAP quando estão não é possível analisar um valor booleano. Inclui um `<attribute-name>` elemento para dar o contexto do erro.  |
|referência do valor-não-ldap-compatível | Este erro é devolvido por agentes de gestão para o LDAP, LDIF e DSML ou ficheiros simples (com o atributo de nome de domínio configurados) quando um valor de nome de domínio não está em conformidade com a especificação de LDAP. Esta mensagem de erro inclui um `<attribute-name>` elemento para dar o contexto do erro.|
|valor inválido de referência | Este erro é devolvido por um agente de gestão, quando um nome de domínio não cumpre as restrições de FIM, que incluem:  <br/>&#176; Um ou mais caracteres que não são permitidas por FIM   <br/>&#176; Um nome único relativo em branco (também conhecido como RDN)  <br/>&#176; Um nome distinto relativo que excede o máximo de FIM  <br/>&#176;  O número de níveis hierárquicos, o nome de domínio excedeu o máximo de FIM                |
|não suportado--tipo de valor | Este erro é devolvido pelo agente de gestão de DSML ou LDIF quando o tipo do valor fornecido no ficheiro é incompatível com o tipo de atributo, incluindo:    <br/>&#176; É atribuído a um valor URI ou o URL para um atributo de cadeia não ou para qualquer palavra-chave reservada como dn, objectclass ou changetype.  <br/>&#176; Um valor de base64 é indicado para o atributo changetype.  <br/>&#176; Um valor de cadeia que contém carateres não ASCII é dado um atributo binário. |

## <a name="synchronization-errors"></a>Erros de sincronização

| Erro               |Descrição      |
|---------|-------------|
| exceção da dll de extensão  | Este erro ocorre se uma extensão de regras faz com que uma excepção. Se ocorrer este erro, observe o \<informações de erro de excepção\> elemento para examinar a pilha de chamadas da exceção. Em alguns casos, o `<rule-error-info>` está presente e fornece informações adicionais sobre de que regra estava a ser processada quando ocorreu o erro.        |
| falhas de dll de extensão | Este erro ocorre quando o processo de executar a extensão de regras inesperadamente terminado. Este erro pode apenas ocorrer quando uma extensão da regra está a ser executada fora do processo. Uma causa possível para este valor de erro é que a extensão de regras está a chamar código que faz com que uma violação de acesso. |
| extensão dll-tempo limite                          | Este erro ocorre se o cliente tiver configurado um tempo limite de extensão e a chamada para um ponto de entrada do código de extensão único cliente excede o limite de tempo configurado. O `<exception-error-info>` proporcionará a nível contextual das informações sobre o ponto de entrada estava a ser chamado quando ultrapassou o tempo limite. Em alguns casos o ' < informações de erro de regra > estarão presentes e irá fornecer informações adicionais sobre os quais regra estava a ser processada quando ocorreu o erro. Tenha em atenção que, quando está a depurar o processo que está a executar a extensão, tempos limite não são impostos.  |              
| Extension-Projection-Object-type-not-set        | Este erro ocorre a implementação do **IMASynchronization.ShouldProjectToMV** método na extensão de regras não especifica o tipo de objeto do metaverso.                    |
| extensão-projecção-inválido--tipo de objeto        | Este erro ocorre quando a implementação do **IMASynchronization.ShouldProjectToMV** método na extensão do regras define o valor do tipo de objeto metaverso de saída para um valor que não está listado no Metaverse Designer de sincronização do Service Manager. Certifique-se de que o método utiliza um dos valores do tipo de objeto especificado.   |
| Extension-JOIN-Resolution-Invalid-Object-type   | Este erro ocorre quando a implementação do **IMASynchronization.ResolveJoinSearch** método na extensão do regras define o valor do tipo de objeto metaverso de saída para um valor que não está listado no Metaverse Designer de sincronização do Service Manager. Certifique-se de que o método define o valor do tipo de objeto metaverso de saída para um objeto listado valores do tipo.   |
| Extension-JOIN-Resolution-Index-Out-of-Bounds   | Este erro ocorre quando uma implementação do **IMASynchronization.ResolveJoinSearch** método na extensão do regras definido um valor de índice que é negativo ou maior ou igual ao número de objetos de metaverso.                     |
| extensão-aprovisionamento-chamada--atingido o limite       | Este erro ocorre quando o **IMASynchronization.Provision** método é chamado mais de 10 vezes durante a sincronização de um único objeto. Este método pode ser chamado mais do que uma vez se a lógica de cliente no método de aprovisionamento deprovisions um objeto e não existe devolução de chamada de atributo que faz com que uma alteração ao objeto de metaverso, causando uma nova chamada para aprovisionar. O limite de 10 chamadas para o método de aprovisionamento está definido para parar possíveis notas aprovisionamento infinita.                                                                                                                                               |
| extensão-desaprovisionamento-inválido-resultado         | Este erro ocorre quando uma implementação do **IMASynchronization.Deprovision** método devolve um valor de enumeração DeprovisionAction inválido. Certifique-se de que o método devolve um valor válido.        |
| extensão de entrada-ponto-não-implementação           | Este erro ocorre quando uma extensão de regras emite um **EntryPointNotImplementedException** exceção. |
| extensão--atributo-valor inesperado            | Este erro ocorre quando uma extensão de regras emite um **UnexpectedDataException** exceção.            |
| flow-multi-values-to-Single-Value              | Este erro ocorre quando uma regra de fluxo de atributos importação ou exportação configurada no Synchronization Service Manager tenta fluxo um atributo com múltiplos valores para um atributo de valor único. Este erro é devolvido apenas para as regras de fluxo direto configuradas no Synchronization Service Manager. Se a regra de fluxo utiliza uma extensão de regras que passa a um atributo de valor único, vários valores de **TooManyValuesException** exceção é emitida.          |
| CS atributo-correspondência                     | Este erro ocorre quando o tipo do atributo importado não corresponde ao tipo de atributo especificado no esquema do agente de gestão. Uma causa deste erro pode ser que o esquema de origem de dados ligada armazenados tornou-se desatualizada com o esquema real da origem de dados ligada. Para colocar o esquema de origem de dados ligada armazenados atualizado, atualize o esquema utilizando o Synchronization Service Manager.|
| JOIN-Object-ID-must-be-Single-Valued            | Este erro ocorre quando a origem de dados utilizado para associar o valor do atributo que um objeto de metaverso através de uma regra de associação especificado nas propriedades de um agente de gestão no Gestor de serviço de sincronização contém mais do que um valor. O valor de atributo de origem de dados utilizado na regra de associação só pode conter um valor único.     |
| DN-índice-out-de-limites       | Este erro ocorre quando o valor de índice de componente de nome único utilizado um fluxo de atributos de importação configurado nas propriedades de um agente de gestão no Gestor de serviço de sincronização for maior do que o número de componentes no nome único do objeto de origem.      |
| regra-violação de conector-filtro                | Este erro ocorre quando efetuar uma adicionar ou mudar o nome de operação de aprovisionamento ou exportar o fluxo de atributos e quando um objecto do conector torna-se de um objeto de seccionador filtrado como resultado de uma configuração de filtro de conetor. Este valor não ocorrer em objetos de conector explícita.    |
| não suportado contentor-eliminação   | O agente de gestão está a tentar eliminar um objeto de contentor durante desaprovisionamento. Os agentes de gestão de FIM não é possível eliminar objetos de contentor com objetos subordinados.            |
| ambiguous-Import-flow-from-Multiple-Connectors   | Este erro ocorre quando tem vários conectores no agente de gestão de origem ligados ao objeto de metaverso e uma regra de fluxo de atributos de importação declarativa está definida. Para importar atributos através de um agente de gestão com vários conectores para um objeto de metaverso, utilize uma extensão de regras para definir as regras de fluxo em vez de configurar uma regra direta nas propriedades de um agente de gestão.                                                                                                                                                                                                          |
| ambiguous-Export-flow-to-Single-Valued-Attribute | Este erro ocorre quando a regra de fluxo de exportação, configurada nas propriedades para um agente de gestão no Gestor do serviço de sincronização, tenta fluxo vários valores de um objeto de metaverso para um atributo de valor único.       |
| Não é possível-análise--id de objeto | O valor de cadeia que é utilizado para pesquisar um objeto de metaverso numa regra de associação especificado nas propriedades de um agente de gestão no Gestor de serviço de sincronização não está no formato correto do identificador exclusivo global (GUID). O formato GUID é {nnnnnnnn-nnnn-nnnnnnnn-nnnnnnnnnnnn} em que n é um número hexadecimal. |
| unexported contentor-mudar o nome  | A implementação do método IMVSynchronization.Provision ou IMASynchronization.Deprovision está a tentar mudar o nome de um objeto de contentor com um ou mais objetos de unexported subordinado.   |
| violação de restrição de mV           | Este erro ocorre quando o fluxo de atributos de importação direta ocorre e o valor de atributo do espaço de conector excede as restrições de comprimento do atributo metaverso. |
| bloqueio-erro-necessidades-repetir   | Múltiplos agentes de gestão estão a tentar sincronizar o mesmo objeto de espaço de conector. Execute novamente o agente de gestão.     |
| violação de índice exclusivo              | Um utilizador é manualmente a definir um índice exclusivo num atributo numa tabela metaverso. Não configure manualmente as tabelas de metaverso.    |
| Perdeu-chave de encriptação                            | Os conjuntos de chaves de encriptação estão em falta no servidor que está a executar o FIM.  |
| Erro inesperado                              | Este erro ocorre quando o motor de sincronização experimenta aplicar uma alteração para o metaverso (incluindo o fluxo de atributos de aprovisionamento e a exportação). Este erro pode apenas ocorrer durante a execução que são aplicadas as alterações para o metaverso. Verifique o registo de eventos para obter mais informações.  |
| exportado-alteração-não-importado novamente                 | Este erro ocorre quando as alterações que são exportadas para um agente de gestão não são reconfirmed durante este agente de gestão de importação executar. Um utilizador ou um processo de sistema operativo fora do FIM mudou os dados da origem de dados ligada de uma forma que indica um problema de configuração em que está a tentar a regra de fluxo de atributos de exportação para o fluxo de um valor para um objeto de origem de dados ligada, mas a origem de dados ligada repõe automaticamente o valor para algo diferente sem reportar um erro para o agente de gestão. O \<importado novamente não de alteração\> elemento indica que as alterações não foram reconfirmed. |
| Não é possível-análise-dn-componente                      | Este erro é devolvido por alguma gestão de agente que tenha um estilo de LDAP nome único (também conhecido como DN configurado) e falha na sincronização a partir do espaço de conector para o metaverso. Não é possível analisar um componente de nome único por um mapeamento de dncomponent porque não está no formato correto para o tipo de atributo de destino.|
| falta-partição-para-execução-passo                 | Este erro indica que não é possível localizar a partição especificada no perfil de execução. Certifique-se de que a partição não foi eliminada ou mudada.  |

## <a name="export-errors"></a>Erros de exportação

| Erro                    |Descrição     |
|--------------------------|------------|
| objeto CD em falta         | Este erro é devolvido quando um modificar de um objeto seja exportado para a origem de dados ligada, mas não é possível encontrar o objeto da origem de dados ligada. É devolvido apenas para os agentes de gestão baseado na chamada. A causa deste erro é que uma pessoa ou processo externo eliminou um objeto de origem de dados ligada fora do FIM.  |
| objeto CD existente        | Este erro é devolvido quando um adicionar seja exportado para a origem de dados ligada, mas o objecto já está presente na origem de dados ligada. É devolvido apenas para os agentes de gestão baseado na chamada e os agentes de gestão de base de dados relacional. |
| âncora duplicado          | Este erro é devolvido se a âncora num objeto recentemente aprovisionada não é exclusiva. É devolvido apenas para chamadas e agentes de gestão de base de dados, bem como o agente de gestão para servidores de diretório Sun e Netscape. Se este erro for detetado, verifique as regras de construção de âncora para se certificar de que foi definido um valor de âncora exclusivo para cada objeto.      |
| atualização ambígua          | Este erro é devolvido quando o agente de gestão não é possível aplicar uma atualização ou eliminar delta porque a âncora não é exclusiva.
